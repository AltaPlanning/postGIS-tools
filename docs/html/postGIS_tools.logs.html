postGIS_tools.logs module¶

Overview of logs.py¶

This is a low-tech solution to leaving a trail of breadcrumbs behind in your PostgreSQL database. It creates a table named db_history and many postGIS_tools.functions use the log_activity() function to update this table after successfully completing their task.

Example¶

>>> import postGIS_tools as pGIS
>>> pGIS.log_activity("my_database", "execute_query", "UPDATE my_table SET my_col = 'my value'", debug=True)
Logging to db_history:
        INSERT INTO db_history (username, function_name, query_text, update_time, user_os, user_cpu)
            VALUES ('aaron', 'execute_query', 'UPDATE my_table SET my_col = ''my value''',
                    '2019-11-23 09:00:26 PST', 'Darwin', 'Aaron-MBP.local');
postGIS_tools.logs.log_activity(db_name: str, function_name: str, query_text: str = '', local_timezone: str = 'US/Pacific', host: str = 'localhost', username: str = 'postgres', password: str = 'this-is-a-placeholder-password', port: int = 5432, debug: bool = True)[source]¶

Record what was done to a database in a new row within the db_history table. User must specify the database and function name that was run. Optionally, the raw SQL code can also be stored in the query_text column.

If the db_history table does not exist yet, make the table before inserting this row.

Parameters

  • db_name – name of the database (string)

  • function_name – the name of the function that was run (string)

  • query_text – the SQL text that was run (string)

  • local_timezone – the user’s local (or preferred) timezone (string). Must match datetime.datetime options

  • host – name of the pgSQL host (string). eg: ‘localhost’ or ‘192.168.1.14’

  • username – valid PostgreSQL database username (string). eg: ‘postgres’

  • password – password for the supplied username (string). eg: ‘mypassword123’

  • port – port number for the PgSQL database. eg: 5432

  • debug – boolean to print messages to console

Returns

inserts a new row into the db_history table
